Registering Your Clients

After you've configured your clients, you can start updating them from your Current server, right? Sorry, but no - you must register your clients with the Current server. This can be done the first time you run up2date or with the rhnreg_ks. Clients running Red Hat Linux 7.3 or earlier should run the rhn_register command.

Follow the directions given by one of the above commands for registration. Note that it really does not matter to Current when username or password you use. The username, password, and most of the information handled in registration is not yet used by the Current server as Current does not do client tracking. However, it does register your client properly.

If you client is already registered with the Red Hat Network you will need to unregister it. This can be done by moving two files out of the way. They are /etc/sysconfig/rhn/up2date-uuid and /etc/sysconfig/rhn/systemid. The Red Hat Network registration does not work with Current.
